# Project Rules

Never regenerate existing modules.

Never duplicate enums.

Never duplicate exceptions.

Never duplicate IDs.

Never create duplicate folders.

Never create duplicate files.

Always inspect the uploaded project before writing code.

Import existing classes instead of redefining them.

Only generate the requested module.

Never modify unrelated modules.

Always generate unit tests.

Always update documentation.

Always perform a self review.

Treat every task as a Pull Request.

The uploaded project is the single source of truth.
